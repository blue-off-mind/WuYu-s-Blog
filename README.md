# Digital Editorial Journal - Technical Documentation

This guide provides comprehensive instructions for manually deploying the Digital Editorial Journal application.

## 1. Project Overview

*   **Type**: Single Page Application (SPA)
*   **Framework**: React 19 + Vite 5
*   **Language**: TypeScript
*   **Styling**: Tailwind CSS 4
*   **State Management**: React Context + Supabase (Realtime)

## 2. Prerequisites

Ensure your build environment meets these requirements:

*   **Node.js**: Version 18.0.0 or higher
*   **Package Manager**: pnpm (recommended), npm, or yarn

## 3. Installation & Build

### Install Dependencies
```bash
# Using pnpm (Recommended)
pnpm install

# Using npm
npm install
```

### Configure Environment Variables
Create a `.env` file in the root directory (based on `.env.example`).
**Required Variables:**
```env
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
```
> **Note:** Variables must be prefixed with `VITE_` to be exposed to the client-side bundle.

### Build for Production
This command compiles TypeScript and bundles the application.
```bash
# Using pnpm
pnpm run build

# Using npm
npm run build
```

## 4. Output Directory

After a successful build, all static files will be generated in the **`dist`** folder.
*   **Path**: `./dist`
*   **Content**: `index.html`, `assets/` (js, css, images), `favicon.svg`

## 5. Deployment Guides

### Option A: Nginx (Linux Server)

Since this is an SPA, you must configure Nginx to redirect all non-file requests to `index.html` so client-side routing works.

**Sample `nginx.conf` Block:**

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # Point to the 'dist' folder generated by the build
    root /var/www/digital-editorial-journal/dist;
    index index.html;

    location / {
        # CRITICAL: Fallback to index.html for SPA routing
        try_files $uri $uri/ /index.html;
    }

    # Optional: Cache static assets
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, no-transform";
    }
}
```

### Option B: Vercel / Netlify / Static Hosting

Most modern static hosts automatically detect Vite projects.

*   **Build Command**: `pnpm run build`
*   **Output Directory**: `dist`

**Vercel Configuration (`vercel.json`)**:
If you need to manually configure routing on Vercel, place this `vercel.json` in the root:

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

## 6. Database Setup (Supabase)

To enable real-time features (likes, comments):
1.  Create a project at [supabase.com](https://supabase.com).
2.  Go to the **SQL Editor** in your dashboard.
3.  Run the contents of the provided `supabase_schema.sql` file.
4.  Run the contents of `update_schema_moderation.sql` to enable moderation logs.
5.  Copy your Project URL and Anon Key into the `.env` file before building.

## 7. Troubleshooting

*   **Blank Page on Refresh**: Ensure your server (Nginx) is configured with `try_files ... /index.html`.
*   **Images Not Loading**: Verify that `referrerPolicy="no-referrer"` is set on images (already handled in code) if using external providers like Unsplash.
*   **Database Error**: Check the console. If you see "PGRST205", you forgot to run the SQL schema migrations.
